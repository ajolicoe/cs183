Functionality

Find a tutor

A user can use the 'find tutor' button to find a tutor of a certain category and subject. They can sort these tutors by Name, or rating. This does not require the user to login.

Be a tutor

A user can use the 'be a tutor' button to register themselves as a tutor, they are then directed to an 'edit profile' page where they can add more information about themselves and add subjects to their profile. This requires the user to be logged in.

Request a session with a tutor

A user can request a session with a tutor when viewing the tutor's profile. This requires the user to be logged in.

Message your tutors

Once the tutor accepts a request, the user and the tutor will be able to send messages to each other and view these conversations.

Rate your tutors

Once the tutor accepts a request, the use will be able to rate the tutor's performance.

Respond to messages via email

When a user sends a message to the tutor, or the tutor sends a message to the student, the message can be emailed to the respective recipient. The recipient of the message will then be able to reply to the message via email, and have it displayed in their conversation with the sender.

Difficulties I ran into

I was unable to find a good solution to passing the sender/recipient id's in the email, so I would know which conversation to add the reply to. I tried creating a custom header, but in the end I just hard-coded it into the subject of the email.

I also tried working on the design of the site, but realized I was spending too much time trying to make it look decent, while some of the functionality still needed work.
